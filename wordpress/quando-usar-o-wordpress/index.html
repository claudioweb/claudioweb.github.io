Wordpress
Atualmente o Wordpress vem crescendo muito no mercado, sendo hoje o CMS mais conhecido mundialmente.

A seguir irei descrever  quando utilizar o Wordpress na sua Empresa ou até mesmo em um Freelancer.

 

Quando usar o Wordpress?


O Wordpress é uma ótima ferramenta para realizar projetos grandes e pequenos como: Blogs, Lojas virtuais, Sites Institucionais, Portais e até mesmo Back-end de Aplicativos, não estou dizendo que o Wordpress é a melhor ferramenta para estes projetos, mas ele sendo utilizado da maneira correta, pode alcançar um desempenho igual ou se igualar  a um projeto realizado com outras tecnologias, por exemplo:

  • Blogs, Sites, e Portais utilizando: (Laravel, Cakephp, Codeignter)

  • Lojas virtuais utilizando: (OpenCart, Magento)


Estes não são os únicos motivos para você decidir criar seu site em Wordpress, o tempo de criação também é um deles, por ser mais rápido e fácil, além de por possuir diversos desenvolvedores atualizando e criando plugins em todo o mundo nos seguintes requisitos:

  • SEO (Yoast SEO) - Irá melhorar e ajudar você a se destacar no Google nas pesquisas Orgânicas;

  • Desenvolvimento (ACF /Advanced Custom Fields) - Agiliza o desenvolvimento do site, criando campos via painel administrativo;

  • Formulários (Contact Form 7) - Você mesmo pode criar seus formulários no site sem a ajuda de um desenvolvedor;

  • Loja virtual (Woocommerce) - Este é o meu favorito, com apenas alguns cliques seu site ou blog se transforma em um E-Commerce ou Vitrine. (Precisa de um desenvolvedor);

  • API (WP REST API) - Transforma seu site em uma API somente com a instalação


Plugins indispensáveis para qualquer desenvolvedor que realmente utiliza ou quer começar a utilizar Wordpress.

 

Quando não usar o Wordpress?


O Wordpress não é perfeito e dúvido muito que um dia será, como desenvolvedor não costumo utilizar Wordpress para projetos grandes que possuem muitos serviços como Webservices e APIs, ou até mesmo Portais com milhares de posts que não irão conseguir suportar as requisições, fora que o site não terá um banco de dados bom para manutenção futura.

Por fim, sempre antes de iniciar um projeto, verifique todos os requisitos acima e o mais importante, verifique se você irá precisar de um Banco de dados e um Core da aplicação bem estruturado para suportar diversos desenvolvedores, aplicações e APIs atuando junto com milhares de requisições de usuários, e se estará disposto a ficar fazendo alterações longas para tentar contornar as funcionalidades padrões do Wordpress (famosa gambiarra).

A melhor forma de fazer seu wordpress suportar um número superior de requisições é utilizando alguns plugins de cache, que economizará algumas requisições básicas do servidor, com isso melhorando a estabilidade e velocidade do seu projeto.

 
Pergunte-se sempre, será que preciso utilizar Wordpress neste trabalho?
